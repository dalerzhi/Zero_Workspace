# 📧 收件人/抄送识别优化

_优化日期：2026-03-09_

---

## ❌ 问题：之前的错误

### 错误场景
```
邮件头：
收件人：丽珍 <lizhen.liu@cheersucloud.com>
抄送：支彬 <zhibin@cheersucloud.com>, 李曼瑶 <...>

邮件内容：
"账单确认无误，可发与客户确认账单，谢谢~"

之前的总结：
## ⚡ 需要行动
- 账单确认无误，可发与客户确认账单 ❌ 错误！这是抄送邮件，不需要我处理
```

### 问题根因
1. **没有解析邮件头** - 不区分 To 和 CC
2. **行动项误提取** - 抄送邮件中的行动词被错误提取
3. **优先级混乱** - 抄送邮件和收件人邮件混在一起

---

## ✅ 优化：智能识别

### 1. 解析邮件头

```python
def is_user_recipient(email, user_email='zhibin@cheersucloud.com'):
    """检查用户是否是收件人（而非仅抄送）"""
    to_field = (email.get('to') or '').lower()
    
    # 检查收件人字段是否包含用户邮箱
    if user_email.lower() in to_field:
        return True
    
    # 也检查用户名（如"支彬"）
    user_names = ['支彬', 'zhibin', '支']
    return any(name in to_field for name in user_names)
```

**效果**：
- ✅ 正确识别 `To: zhibin@...` → 收件人
- ✅ 正确识别 `CC: zhibin@...` → 抄送
- ✅ 支持中文名识别（"支彬"）

### 2. 分类逻辑升级

**之前**：只看邮件内容关键词
```
内容包含"请确认" → 待办事项 ❌ 可能是抄送邮件
```

**现在**：先看收件人/抄送，再看内容
```python
if not is_to:
    # 抄送邮件 - 通常仅需知悉
    return 'cc_only', '抄送知悉'
else:
    # 收件人邮件 - 可能需要处理
    if action_to_me_score >= 1:
        return 'action', '待办事项'
```

### 3. 行动项归属识别

**明确指派给"我"的模式**：
```python
action_to_me_patterns = [
    r'请 [您你] 确认', r'请 [您你] 核对', r'请 [您你] 审批',
    r'请贵司', r'请贵部门', r'盼复', r'望确认'
]
```

**效果**：
- ✅ "请**您**确认" → 我的任务
- ✅ "请**贵司**提供" → 我的任务
- ❌ "请丽珍确认" → 别人的任务（不提取）
- ❌ 抄送邮件中的"请确认" → 不提取
