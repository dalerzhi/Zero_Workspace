## 🔁 下午补充（Gateway / Feishu 插件）

### Gateway service 回归问题（OpenClaw 2026.3.8）
- 用户反馈：从 `2026.3.2` 升级到 `2026.3.8` 后，Gateway service 管理明显不稳定。
- 重要结论：主会话/agent 侧触发 `openclaw gateway restart` 有概率把 LaunchAgent 服务链路打散，表现为：
  - `openclaw gateway start` 报 `service not installed`
  - 但前台 `openclaw gateway` 裸跑仍可临时工作
- 用户多次通过手动执行 `openclaw gateway install` 才恢复 service。
- 临时规约：**不要由我主动执行 Gateway restart**；需要重启时，由用户手动执行并现场观察。
- 用户在重装 Gateway service 后重启 mac，确认 Gateway 可随系统自动拉起。

### Feishu 插件问题修复进展
- 已修复/缓解：
  - 联系人权限缺失导致的 `contact:*` scope 错误刷日志：在 `~/.openclaw/extensions/feishu/src/bot.ts` 中加入 sender lookup 权限失败抑制窗口，避免每条消息都打联系人 API。
  - 用户随后也在飞书后台补了机器人权限；重启后未再看到 `contact:contact.base:readonly` 类报错。
- duplicate plugin id 根因进一步收敛：
  - 不是两个不同插件冲突，而是 **同一个 `~/.openclaw/extensions/feishu` 目录被两套发现机制重复加载**。
  - 重复来源是：`openclaw.plugin.json` + `package.json` 中的 `openclaw.extensions -> ./index.ts`。
- 本地 workaround：
  - 修改 `~/.openclaw/extensions/feishu/package.json`
  - 删除 `openclaw.extensions` 字段，保留 `openclaw.plugin.json` 作为插件入口声明
- 风险：该 workaround 在将来插件 update/install 时可能被覆盖，导致 duplicate plugin id 问题复发。
