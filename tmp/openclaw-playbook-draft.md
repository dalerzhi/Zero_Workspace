# OpenClaw 玩法手册：让 AI 真正为你干活

---

## 开场：不是陪聊，是替你干活

市面上 AI 助手很多，但大多数止步于"对话"。你问它答，你让它写，它给你一段文字——然后呢？然后你还得自己复制粘贴、自己执行、自己跟进。

OpenClaw 不一样。它的核心设计理念是：**Agent 应该能替你动手**。

不是"帮我写一个脚本"，而是"帮我写完、跑完、把结果发我邮箱"。

不是"告诉我这个 bug 怎么修"，而是"帮我把这个 bug 修完，PR 已经开好了"。

这种"闭环能力"来自三个设计：

- **工具调用**：它能执行命令、读写文件、调用 API、控制浏览器
- **会话持久化**：它有记忆，知道你的偏好、项目上下文、历史决策
- **子 Agent 架构**：复杂任务可以拆分，多线并行，互不干扰

下面这 10 个玩法，就是围绕这个"替你干活"的核心理念展开的。

## 玩法 1：AI 自动修 GitHub Bug

**一句话亮点**：丢一个 issue，回来检查时 PR 已经开好了。

**适合谁**：独立开发者、开源项目维护者、技术团队 Leader

**真实场景**：

你维护着一个开源项目，Issue 列表里积压了十几个 bug 标签。人力有限，不可能一个个手动处理。你只想让 AI 帮你筛选出"能修的"，修完之后给你一个 PR 链接，你来 Review 就行。

**快速上手**：

```bash
# 拉取你项目里 label=bug 的 issue，让 AI 自动修
/gh-issues yourname/yourrepo --label bug --limit 5
```

它会自动：
1. 分析每个 issue 的可修复性
2. 创建分支、写代码、跑测试
3. 提交 PR，并在 PR 里附上改动说明

如果你有 Fork 仓库的习惯，还可以指定 `--fork`，让所有改动都在你的 Fork 上进行，干净利落。

## 玩法 2：多子 Agent 并行编码

**一句话亮点**：一个主 Agent 指挥，多个子 Agent 同时干活。

**适合谁**：需要同时处理多个编码任务的开发者

**真实场景**：

你今天有三个任务要处理：重构一个模块、修复两个 bug、写完一个新的 API endpoint。单线程做要一天，但你希望"同时推进"。

**快速上手**：

OpenClaw 支持用 git worktree 把代码库拆成多个独立工作区，每个子 Agent 在自己的 worktree 里干活，互不干扰：

```bash
# 为三个任务分别创建 worktree
git worktree add -b fix/issue-78 /tmp/issue-78 main
git worktree add -b fix/issue-99 /tmp/issue-99 main
git worktree add -b feat/api-v2 /tmp/api-v2 main

# 启动三个并行的编码 Agent
# (在 OpenClaw 里自然语言描述即可，它会自动调度)
```

你可以一边开会，一边让三个 Agent 并行推进。回来时，三个分支的代码都写好了。

## 玩法 3：飞书语音助手

**一句话亮点**：让它说的话，变成飞书里可以播放的语音消息。

**适合谁**：经常用飞书沟通、需要语音播报内容的用户

**真实场景**：

你在开车，不想看手机屏幕，但有一份重要文档需要快速了解。你让 OpenClaw 把文档总结一下，用语音发到你飞书里——不是发一个音频文件，而是原生的语音气泡，点一下就能播放。

**快速上手**：

OpenClaw 支持将文本转成 Opus 格式的语音消息，符合飞书原生语音消息的格式要求：

```bash
# 生成语音并飞书发送（一行搞定）
say -v Ting-Ting "这是测试语音" | ffmpeg -f wav -i pipe:0 -c libopus voice.opus -y
openclaw message send --channel feishu --target <用户ID> --media voice.opus
```

你可以让它帮你读邮件摘要、读日程安排、读代码审查意见——任何文字内容都能变成语音。

## 玩法 4：飞书工作流总管

**一句话亮点**：查日程、读消息、建文档、发通知，一条龙搞定。

**适合谁**：飞书重度用户、需要管理复杂工作流的人

**真实场景**：

周一早上，你想快速了解这周的安排。OpenClaw 可以帮你：
- 拉取飞书日历里这周的所有会议
- 检查有哪些待办任务快到期了
- 把重要事项整理成一份文档
- 发到指定群里，或者私信给你

**快速上手**：

OpenClaw 内置了完整的飞书 API 技能集：

```bash
# 查看本周日程
"帮我看看这周有哪些会议"

# 创建飞书文档
"帮我创建一份周报文档，标题是'第12周工作总结'"

# 搜索飞书消息
"帮我搜一下上周群里有没有提到'上线'"
```

你可以把这些组合成一条自然语言指令，OpenClaw 会自动拆解执行。

## 玩法 5：Heartbeat + Cron 后台分身

**一句话亮点**：你不在的时候，它在默默干活。

**适合谁**：需要定时任务、后台监控、自动化流程的用户

**真实场景**：

你设置了每天早上 6 点自动生成一份"昨日工作总结"——包括 git 提交、邮件处理、学习记录。你醒来时，这份总结已经躺在你的飞书里了。

**快速上手**：

OpenClaw 有两种后台机制：

- **Heartbeat**：每隔一段时间（比如 30 分钟）主动检查一次，看有没有需要处理的事情
- **Cron**：精确时间点的定时任务，比如"每天早上 6 点运行"

```bash
# 添加一个定时任务
openclaw cron add --name daily-summary --schedule "0 6 * * *" --command "./scripts/daily-summary.sh"
```

Heartbeat 更灵活，适合"有事儿就干"的场景；Cron 更精确，适合"定时执行"的场景。两者结合，你就有了一个 24 小时不休息的分身。

## 玩法 6：浏览器自动化

**一句话亮点**：它能像人一样操作浏览器，不只是爬网页。

**适合谁**：需要频繁操作网页、填写表单、抓取动态内容的用户

**真实场景**：

你每天都要登录某个系统，下载一份报告，然后上传到另一个系统。这是一个无聊且重复的操作，但因为没有 API，只能手动做。OpenClaw 可以帮你"看见"网页、点击按钮、填写表单——就像一个真人在操作。

**快速上手**：

OpenClaw 支持 Playwright 驱动的浏览器自动化：

```bash
# 打开一个网页，截图
browser action:open url:"https://example.com"

# 点击某个元素
browser action:act kind:click selector:"button.submit"

# 填写表单
browser action:act kind:fill selector:"input[name='email']" text:"your@email.com"
```

你可以用它做更复杂的事情：自动化测试、批量截图、价格监控、竞品分析——任何需要"看见网页"的场景。

## 玩法 7：技能商店 / ClawHub

**一句话亮点**：自己写技能太累？去技能商店下载现成的。

**适合谁**：不想重复造轮子、想快速扩展能力的用户

**真实场景**：

你需要一个"发送 Slack 消息"的功能，但不想自己写 API 调用。你去 ClawHub 搜了一下，发现已经有现成的技能了——安装、配置 API Key，两分钟搞定。

**快速上手**：

```bash
# 搜索技能
clawhub search "slack"

# 安装技能
clawhub install slack-notifier

# 查看已安装技能
clawhub list

# 更新技能
clawhub update slack-notifier
```

ClawHub 是 OpenClaw 的技能市场，类似于 VS Code 的插件商店。社区贡献的技能覆盖了各种场景：数据库操作、云服务集成、通知推送……你也可以把自己写的技能发布上去。

## 玩法 8：URL / 视频 / 文件总结

**一句话亮点**：扔一个链接，几秒钟后给你精炼摘要。

**适合谁**：信息过载、需要快速筛选内容的用户

**真实场景**：

你收藏了一堆 YouTube 视频和长文章，但根本没时间看。OpenClaw 可以帮你快速提取核心内容：视频提取字幕并总结、文章提取正文并提炼要点。

**快速上手**：

```bash
# 总结一个网页
summarize "https://example.com/long-article"

# 总结一个 YouTube 视频（自动提取字幕）
summarize "https://youtu.be/xxxxx" --youtube auto

# 总结一个本地 PDF
summarize "/path/to/report.pdf"
```

你还可以指定总结长度、输出格式，甚至只提取字幕不做总结——适合你想快速扫一眼视频内容再决定看不看。

## 玩法 9：AI 图片生成

**一句话亮点**：用 Gemini 3 Pro Image 生成和编辑图片。

**适合谁**：需要快速产出视觉内容的设计师、运营、内容创作者

**真实场景**：

你需要一张博客封面图，描述一下想要的风格，几秒钟后图片就生成了。或者你有一张图，想让 AI 帮你把背景换成夕阳——这也是支持的。

**快速上手**：

```bash
# 生成图片
uv run scripts/generate_image.py --prompt "一只穿宇航服的猫在月球上" --filename "cat.png" --resolution 2K

# 编辑图片（图生图）
uv run scripts/generate_image.py --prompt "把背景换成赛博朋克城市" --filename "output.png" -i input.png

# 多图合成
uv run scripts/generate_image.py --prompt "把这些元素组合成一张海报" -i img1.png -i img2.png -i img3.png
```

支持 1K/2K/4K 分辨率，支持多种宽高比——头像、海报、横幅都能搞定。

## 玩法 10：系统安全巡检

**一句话亮点**：让 AI 帮你做安全审计，而不是自己查文档。

**适合谁**：关心系统安全、需要定期巡检的用户

**真实场景**：

你在一台云服务器上部署了 OpenClaw，担心开放端口太多、SSH 配置不安全。你让 OpenClaw 跑一个安全审计，它告诉你哪些端口暴露了、哪些配置可以加固、建议怎么改。

**快速上手**：

```bash
# 运行安全审计
openclaw security audit --deep

# 应用安全加固建议
openclaw security audit --fix
```

OpenClaw 会检查：
- 开放的端口和服务
- 防火墙配置
- SSH 加固建议
- 磁盘加密状态
- 自动更新状态

你可以把安全审计设为定期任务，每周自动跑一次，有问题及时告警。

## 结尾：先试哪三个？

如果你刚接触 OpenClaw，我建议从这三个玩法开始：

1. **URL / 视频总结**：最直接、最立竿见影。扔一个 YouTube 链接，几秒后拿到核心内容，体验感拉满。

2. **飞书工作流总管**：如果你日常用飞书，这个能立刻感受到"AI 替你干活"的快感——查日程、发消息、建文档，一句话的事。

3. **AI 自动修 GitHub Bug**：如果你是开发者，这个最能体现 OpenClaw 的"闭环能力"。看着 AI 自己开 PR，比自己写代码还爽。

这三个玩通了，其他的自然水到渠成。
